# 第一部分：和弦 -> note_number 数组

MIDI 二进制落地字段（本文件统一）：
- `status`: `note_on=0x90+channel`, `note_off=0x80+channel`
- `channel`: `0-15`
- `note_number`: `0-127`
- `velocity`: `0-127`
- `delta_time(tick)`: 事件相对时间

和弦定义（半音偏移数组 `offsets`）：
- Major triad: `[0, 4, 7]`
- Minor triad: `[0, 3, 7]`
- Diminished triad: `[0, 3, 6]`
- Augmented triad: `[0, 4, 8]`
- Sus2: `[0, 2, 7]`
- Sus4: `[0, 5, 7]`
- Power5: `[0, 7]`
- Major6: `[0, 4, 7, 9]`
- Minor6: `[0, 3, 7, 9]`
- Dominant7: `[0, 4, 7, 10]`
- Major7: `[0, 4, 7, 11]`
- Minor7: `[0, 3, 7, 10]`
- MinorMajor7: `[0, 3, 7, 11]`
- HalfDiminished7 (m7b5): `[0, 3, 6, 10]`
- Diminished7: `[0, 3, 6, 9]`
- Augmented7: `[0, 4, 8, 10]`
- AugmentedMajor7: `[0, 4, 8, 11]`
- Add9: `[0, 4, 7, 14]`
- MinorAdd9: `[0, 3, 7, 14]`
- Add11: `[0, 4, 7, 17]`
- MinorAdd11: `[0, 3, 7, 17]`
- Add13: `[0, 4, 7, 21]`
- 6/9: `[0, 4, 7, 9, 14]`
- m6/9: `[0, 3, 7, 9, 14]`
- Dominant9: `[0, 4, 7, 10, 14]`
- Major9: `[0, 4, 7, 11, 14]`
- Minor9: `[0, 3, 7, 10, 14]`
- Dominant11: `[0, 4, 7, 10, 14, 17]`
- Major11: `[0, 4, 7, 11, 14, 17]`
- Minor11: `[0, 3, 7, 10, 14, 17]`
- Dominant13: `[0, 4, 7, 10, 14, 17, 21]`
- Major13: `[0, 4, 7, 11, 14, 17, 21]`
- Minor13: `[0, 3, 7, 10, 14, 17, 21]`
- 7b5: `[0, 4, 6, 10]`
- 7#5: `[0, 4, 8, 10]`
- 7b9: `[0, 4, 7, 10, 13]`
- 7#9: `[0, 4, 7, 10, 15]`
- 7sus4: `[0, 5, 7, 10]`
- Maj7#11: `[0, 4, 7, 11, 18]`

通用计算：
- 给定根音 `root_note_number = R`
- 和弦音集合：`chord_notes = [R + o for o in offsets]`
- 转位/八度重排：`R + o + 12*k`（`k` 为整数，且结果需在 `0-127`）

MIDI 事件模板（每个和弦都可直接二进制输出）：
```text
for n in chord_notes:
  emit note_on(channel, n, velocity, delta_time=start_tick)
for n in chord_notes:
  emit note_off(channel, n, 0, delta_time=end_tick)
```

# 第二部分：调内和弦级数 -> note_number 计算公式

大调级数根音公式：
- 大调音阶半音偏移：`major_scale = [0, 2, 4, 5, 7, 9, 11]`
- 调中心 `tonic = T`
- 第 `d` 级根音：`degree_root[d] = T + major_scale[d]`

大调三和弦类型模式（I 到 vii°）：
- `qualities = [maj, min, min, maj, maj, min, dim]`

大调七和弦类型模式（I 到 viiø7）：
- `qualities7 = [maj7, min7, min7, maj7, dom7, min7, m7b5]`

C 大调实例（`T=60`，即 C4=60）：
- I: C major, 根音 `60`, 和弦 `[60, 64, 67]`
- ii: D minor, 根音 `62`, 和弦 `[62, 65, 69]`
- iii: E minor, 根音 `64`, 和弦 `[64, 67, 71]`
- IV: F major, 根音 `65`, 和弦 `[65, 69, 72]`
- V: G major, 根音 `67`, 和弦 `[67, 71, 74]`
- vi: A minor, 根音 `69`, 和弦 `[69, 72, 76]`
- vii°: B diminished, 根音 `71`, 和弦 `[71, 74, 77]`

可选：自然小调级数根音（用于借用和弦）
- `minor_scale = [0, 2, 3, 5, 7, 8, 10]`
- `minor_qualities = [min, dim, maj, min, min, maj, maj]`

# 第三部分：功能和声 -> 和弦分组

功能编码（bitmask）：
- Tonic (T): `001`
- Subdominant (S): `010`
- Dominant (D): `100`

C 大调功能映射（每个实体都包含 MIDI 根音和音集）：
- I: `function=001(T)`, `root=60`, `notes=[60,64,67]`
- iii: `function=001(T)`, `root=64`, `notes=[64,67,71]`
- vi: `function=001(T)`, `root=69`, `notes=[69,72,76]`
- ii: `function=010(S)`, `root=62`, `notes=[62,65,69]`
- IV: `function=010(S)`, `root=65`, `notes=[65,69,72]`
- V: `function=100(D)`, `root=67`, `notes=[67,71,74]`
- vii°: `function=100(D)`, `root=71`, `notes=[71,74,77]`

二进制决策规则（编曲器可直接执行）：
- 稳定收束：优先 `D(100) -> T(001)`
- 扩展过渡：`T(001) -> S(010) -> D(100) -> T(001)`
- 每步输出都可映射为 `note_on/off + delta_time + channel + velocity`

# 第五部分：Voice Leading -> note_number 移动规则

输入：前一和弦 `A=[a1,a2,...]`，后一和弦候选转位 `B=[b1,b2,...]`（都为 `note_number`）

1) 最小移动原则（核心目标函数）
- `move_cost(A,B) = sum_i(abs(bi - ai))`
- 在所有 `B` 的八度展开（`bi+12k`）中选 `move_cost` 最小值
- 实用阈值：`avg_move = move_cost / voice_count <= 2` 半音优先

2) 共同音保持
- 若 `x` 同时在 A 与 B，优先 `x -> x`（`delta=0`）
- 该规则直接减少 `move_cost`

3) 平行五度禁止（量化检测）
- 对任意声部对 `(i,j)`：
- `int_prev = (aj - ai) mod 12`
- `int_next = (bj - bi) mod 12`
- `di = bi - ai`, `dj = bj - aj`
- 若 `int_prev==7` 且 `int_next==7` 且 `sign(di)==sign(dj)` 且 `di!=0` 且 `dj!=0`，则记一次违规
- `parallel5_penalty = violations * 100`（可直接加到损失函数）

4) 声部交叉限制
- 对升序排好的声部要求：`b1 <= b2 <= b3 <= b4`
- 若违反则丢弃该候选

5) 最终选择分数
- `score = move_cost + parallel5_penalty + crossing_penalty`
- 取 `score` 最小的 `B`

输出 MIDI：
- 由 `A` 到 `B` 时，对每个声部生成 `note_off(A_i)` 与 `note_on(B_i)`，并写入对应 `delta_time`

# 第六部分：张力与释放 -> 音程协和度量化

音程类定义：`ic = abs(n1 - n2) mod 12`

协和/不协和（硬约束来自音程半音）：
- 协和：`0(纯一), 12~0(纯八), 7(纯五), 5(纯四)`
- 不协和：`1(小二), 11(大七), 6(三全音)`

张力权重（可调，越大越紧张）：
- `w = {0:0, 5:1, 7:1, 3:2, 4:2, 8:3, 9:3, 2:4, 10:4, 1:6, 11:6, 6:7}`

和弦张力值：
- 设和弦音集合 `C=[n1...nk]`
- `tension(C) = (sum_{i<j} w[abs(ni-nj)%12]) / (k*(k-1)/2)`

进行的释放量：
- `release(A->B) = tension(A) - tension(B)`
- `release > 0` 表示释放；`release < 0` 表示继续累积张力

典型 D->T 量化（C 大调）：
- `G7 = [67,71,74,77]`, `C = [60,64,67]`
- 关键导向：`71(B)->72(C)` 为 `+1` 半音，`77(F)->76(E)` 为 `-1` 半音
- 在 MIDI 上即两个 `note_number` 的最短半音级移动，形成高释放概率

附加可控维度（仍是 MIDI 二进制）：
- `velocity`：同和弦提高力度可提升主观张力
- `register(note_number)`：高音区（例如 >72）通常提高张力
- `pitch_bend(0xE0)`：短时偏移增加不稳定性后回中可制造释放
- `control_change(0xB0)`：如 `CC11 Expression` 上扬制造推力，回落制造释放
