# 编曲理论与实践（MIDI 二进制映射版）

## 第一部分：MIDI 轨道与乐器分配 -> channel + program_change

### 1.1 二进制基准
- 统一时基：`PPQ=480`，`4/4` 下每小节 `1920 ticks`。
- `note_on`: 状态字节 `0x90 + channel`，数据字节1=`note_number(0-127)`，数据字节2=`velocity(0-127)`。
- `note_off`: 状态字节 `0x80 + channel`（或 `0x90 + channel, velocity=0`）。
- `program_change`: 状态字节 `0xC0 + channel`，数据字节1=`program(0-127)`。
- `control_change`: 状态字节 `0xB0 + channel`，数据字节1=`CC编号`，数据字节2=`值(0-127)`。
- `pitch_bend`: 状态字节 `0xE0 + channel`，数据字节1=`LSB(0-127)`，数据字节2=`MSB(0-127)`，14-bit 值 `0-16383`，中心值 `8192`。

## 第二部分：段落结构 -> MIDI 轨道组合矩阵

### 2.1 8bit 通道位图定义
- 位序：`b7 b6 b5 b4 b3 b2 b1 b0 = ch9 ch6 ch5 ch4 ch3 ch2 ch1 ch0`
- `1=active`，`0=muted`

### 2.2 段落轨道激活矩阵

| 段落 | 激活 channel | 8bit 位图 | 速度策略（velocity） | 音符密度策略（notes/bar） |
|---|---|---|---|---|
| Intro | 0,9 | `0b10000001` | Piano 55-75, Drums 50-70 | 4-8 |
| Verse | 0,1,2,9 | `0b10000111` | Bass/Guitar 提升到 60-90 | 8-16 |
| Pre-Chorus | 0,1,2,3,9 | `0b10001111` | Strings 新增 50-75 | 12-20 |
| Chorus | 0,1,2,3,4,5,6,9 | `0b11111111` | 全轨 +10~+18 velocity | 20-36 |
| Bridge | 0,3,6 | `0b01001001` | 回落到 45-78 | 6-12 |
| Outro-A | 0,3,6,9 | `0b11001001` | Drums 低力度 45-60 | 6-10 |
| Outro-B | 0,6 | `0b01000001` | 继续衰减 40-65 | 4-8 |
| Outro-End | 0 | `0b00000001` | Piano 35-55 | 1-4 |
