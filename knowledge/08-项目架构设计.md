# 第一部分：项目概述

- 项目名称: `logic-pro-audio-arranger`（重写版）
- 一句话描述: 用乐理栅栏约束 LLM，将主旋律 MIDI 自动编曲为完整多轨 MIDI。
- 技术栈: Python 3.12+ / uv / mido / pretty_midi / basic-pitch / chromadb / anthropic SDK
- 输入: 主旋律 MIDI 文件 或 mp3/wav 音频
- 输出: 多轨 MIDI 文件（可导入 Logic Pro / Ableton / 任意 DAW）

核心设计原则（可运行落地版）：
- MIDI 二进制是唯一“事实数据层”：内部统一为 `note_number/velocity/start_tick/duration_tick/channel`，导出时再映射为 `note_on/note_off + delta_time`。
- 乐理作为栅栏（Guardrails）：LLM 只能在合法调式、和声、音域、节奏网格内生成音符。
- LLM 只产出结构化 JSON（tool_use），不直接产出自由文本 MIDI 指令。
- 先检索后生成：RAG 先提供可复用和弦/鼓组/低音 pattern，再由 LLM 组合与变体。
- 双重校验：生成后必须通过栅栏验证器，不合法音符自动修正到最近合法音。

运行形态：
- CLI 主入口：`uv run arranger arrange --input melody.mid --style pop --out arrangement.mid`
- 任务编排顺序：`input -> analysis -> guardrails -> rag -> llm -> validate -> midi assemble`

# 第二部分：系统架构图（ASCII）

```text
[Input: melody.mid / audio.mp3]
       ↓
[Audio→MIDI 模块] (basic-pitch / demucs optional)
       ↓
[旋律分析模块] → key, tempo, time_sig, chord_hints
       ↓
[栅栏生成器] → 合法 note_number/velocity/tick 范围
       ↓
[RAG 检索] ← ChromaDB (乐理知识库)
       ↓
[LLM 编曲引擎] (Claude API)
  ├→ 和弦进行生成
  ├→ 鼓组 pattern 选择/变化
  ├→ Bass line 生成
  ├→ Piano/Keys comping
  └→ String/Pad 铺垫
       ↓
[栅栏验证器] → 检查每个音符是否合法
       ↓
[MIDI 组装器] → 多轨 MIDI 文件
       ↓
[Output: arrangement.mid]
```

MIDI 二进制核心数据流（贯穿全链路）：
- 输入标准化后统一为 `Note[]`。
- `Note[]` 在每个模块内只做“合法变换”，不做语义漂移。
- 最终导出阶段才做 `ticks -> delta_time` 编码并写入 `.mid` 文件。

模块职责与调用关系：
- `audio`: 仅负责输入统一化（音频转 MIDI），不参与和声决策。
- `analysis`: 产出结构化分析结果 `key/tempo/time_sig/sections/chord_hints`。
- `guardrails`: 由分析结果生成可执行约束，并提供修正器。
- `rag`: 从知识库中返回 pattern 候选，作为 LLM 上下文。
- `engine`: 调用 Claude tool_use 生成各轨素材并组装为 `Arrangement`。
- `midi`: 把 `Arrangement` 转为标准 SMF 多轨 MIDI。
- `models`: 统一跨模块的数据协议，保证类型可校验、可序列化。

# 第四部分：核心数据模型（Pydantic）

```python
from pydantic import BaseModel, Field


class Note(BaseModel):
    note_number: int = Field(ge=0, le=127)
    velocity: int = Field(ge=0, le=127)
    start_tick: int = Field(ge=0)
    duration_tick: int = Field(gt=0)
    channel: int = Field(ge=0, le=15)


class Pattern(BaseModel):
    type: str
    notes: list[Note]
    tempo: int = Field(gt=20, lt=300)
    time_sig: str  # e.g. "4/4"
    key: str       # e.g. "C_major"
    tags: list[str] = []


class Arrangement(BaseModel):
    tracks: dict[str, list[Note]]
    metadata: dict[str, str | int | float]


class GuardrailSet(BaseModel):
    allowed_notes: set[int]               # 允许的 pitch class / note_number
    allowed_velocities: tuple[int, int]   # (min_v, max_v)
    tick_grid: int                        # 例如 120 (16th @ PPQ480)
    note_ranges: dict[str, tuple[int, int]]  # track_name -> (low, high)
```

实现要点：
- `Note` 是全系统最小原子，所有模块只读写 `Note`。
- `Pattern` 用于 RAG 和鼓组/和声复用，便于跨段落变体。
- `Arrangement` 是导出前最终容器。
- `GuardrailSet` 由 analysis+knowledge 推导，验证器只依赖该模型执行硬约束。

# 第五部分：LLM 编曲流程详细设计

Step 1: 旋律分析 -> 结构化数据
- 从主旋律 MIDI 提取：`key`, `tempo`, `time_sig`, `phrase_boundaries`, `chord_hints`, `melody_density`。

Step 2: 生成栅栏集
- 依据调性和段落生成 `GuardrailSet`：
  - `allowed_notes`（调内/和弦内优先）
  - `allowed_velocities`（按段落能量分层）
  - `tick_grid`（如 120 tick）
  - `note_ranges`（Bass/Piano/Pad/Lead 各自音域）

Step 3: RAG 检索参考 patterns
- 在 ChromaDB 中按 `genre + section + key + tempo` 检索：
  - 和弦进行候选
  - 鼓组 pattern 候选
  - Bass / Comping 参考片段

Step 4: LLM 调用（使用 Claude tool_use）
- Tool 1: `generate_chord_progression(key, mood, section) -> note_number[][]`
- Tool 2: `select_drum_pattern(genre, section, energy) -> Pattern`
- Tool 3: `generate_bass_line(chords, genre) -> note_number[] + tick[]`
- Tool 4: `generate_comp(chords, instrument, genre) -> note_number[][] + tick[]`

建议工具定义（`engine/tools.py`）：
```python
TOOLS = [
    {
        "name": "generate_chord_progression",
        "input_schema": {
            "type": "object",
            "properties": {
                "key": {"type": "string"},
                "mood": {"type": "string"},
                "section": {"type": "string"}
            },
            "required": ["key", "mood", "section"]
        }
    },
    {
        "name": "select_drum_pattern",
        "input_schema": {
            "type": "object",
            "properties": {
                "genre": {"type": "string"},
                "section": {"type": "string"},
                "energy": {"type": "number"}
            },
            "required": ["genre", "section", "energy"]
        }
    },
]
```

Step 5: 栅栏验证（不合法的音符 -> 修正到最近合法音）
- 校验项：
  - `note_number` 是否在调内/和弦内/音域内
  - `velocity` 是否在允许区间
  - `start_tick/duration_tick` 是否对齐 `tick_grid`
- 修正规则：
  - 音高：投影到最近合法音（最小半音距离）
  - 力度：clip 到 `[min_v, max_v]`
  - 节奏：量化到最近网格点

Step 6: 组装 MIDI
- 将各轨 `Note[]` 转 SMF 事件：
  - `note_on(channel, note_number, velocity)`
  - `note_off(channel, note_number, 0)`
  - 按 `start_tick` 排序后转换 `delta_time`
- 写出 `arrangement.mid`，并附带 tempo/time signature/meta track。

# 第七部分：MVP 实现路线图

- Phase 1: MIDI 读写 + 栅栏验证器
  - 交付: `midi/parser.py`, `midi/builder.py`, `guardrails/validator.py`
  - 验收: 输入旋律 MIDI 后可输出合法单轨修正版 MIDI

- Phase 2: 乐理 RAG 数据库 + 知识导入
  - 交付: `rag/seed.py` 将 `knowledge/*.md` 结构化入库
  - 验收: 可按 `genre/section/key` 检索出 pattern top-k

- Phase 3: LLM 编曲引擎（Claude tool_use）
  - 交付: `engine/llm.py`, `engine/tools.py`, `engine/arrange.py`
  - 验收: 可生成 chords + drums + bass + comp 四类轨道 JSON

- Phase 4: 音频→MIDI pipeline
  - 交付: `audio/transcribe.py`（basic-pitch），`audio/separate.py`（可选）
  - 验收: 输入 mp3/wav 可自动转旋律 MIDI 并接入编曲流程

- Phase 5: CLI + Web UI
  - 交付: `cli.py` 完整命令，后续可接 FastAPI/Gradio Web 界面
  - 验收: 一条命令完成端到端：`input -> arrangement.mid`

MVP 成功标准：
- 不依赖 Node/Go/Java。
- 以 Python + uv 单栈运行。
- 全流程以 MIDI 二进制参数为核心数据。
- 生成结果可直接导入 Logic Pro/Ableton 并播放。
